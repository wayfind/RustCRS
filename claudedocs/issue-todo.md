#### ISSUE-UI-004 - GET /admin/tags 返回 405 Method Not Allowed

**优先级**: P1
**模块**: 管理后台/API Keys/标签管理
**状态**: 🔴 待修复
**发现时间**: 2025-11-03
**发现方式**: UI 深度漫游测试（Playwright）

**重现步骤**:
1. 登录管理后台
2. 点击 "API Keys" → "+ 创建新 Key"
3. 或点击某个 Key 的 "编辑" 按钮
4. 观察浏览器控制台错误

**预期行为**:
- GET /admin/tags 返回 HTTP 200
- 返回已有标签列表用于下拉选择

**实际行为**:
- 返回 HTTP 405 Method Not Allowed
- 标签下拉列表无法加载已有标签

**错误信息**:
```
Failed to load resource: the server responded with a status of 405 (Method Not Allowed)
API GET Error: Error: HTTP 405: Method Not Allowed
获取标签失败: Error: HTTP 405: Method Not Allowed
```

**🔍 根因分析**:
- **根本原因**: /admin/tags 端点未实现 GET 方法
  - 为什么 1: 前端请求 GET /admin/tags 失败
  - 为什么 2: 后端返回 405 而非 404，说明路由存在但方法不支持
  - 为什么 3: 可能只实现了 POST/PUT/DELETE 方法，未实现 GET
  - 为什么 4: 标签管理功能迁移时未完整实现所有方法
  - 为什么 5: **Node.js→Rust 迁移时未完整迁移标签管理的所有接口**
- **根因类型**: 📚 缺失功能（部分实现）
- **依赖问题**: 无
- **阻塞问题**: 无
- **影响范围**:
  - 创建/编辑 API Key 时无法选择已有标签
  - 只能手动输入新标签，无法复用

**集成测试名称**: `test_get_tags_endpoint`

**备注**:
- 标签可以手动创建，但无法列表化管理
- 需要实现 GET /admin/tags 返回所有标签的列表

- **阻塞问题**: 无
- **影响范围**:

  - 可能影响其他日期字段（最后使用时间、过期时间等）

**集成测试名称**: `test_api_key_date_format`

**备注**:
- 需要检查后端返回的日期格式
- 前端可能期望 ISO 8601 格式或 Unix 时间戳
